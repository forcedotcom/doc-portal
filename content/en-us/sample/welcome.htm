Welcome!

If you are reading this in your own portal, congratulations, it’s working!

If you are reading this locally, start with the README file in the github repository for instructions on how to get the portal up and running first.

What now?

Great question.

This is one of the default topics included as sample content in the portal. More than anything it’s designed to give you reassurance that the portal is working. Why not start by replacing it with your own content?

There’s a topic on replacing the default topics with your own content. A link to it is at the bottom of this topic, but instead of looking for that, why not test search?

That search link illustrates something nice about this portal - everything is a bookmarkable link, including searches. Feel free to bookmark them, give them to your users, etc.

Once you’re gotten your own content in the portal, you probably want to change the styling of the portal. Perhaps the header color is just the wrong shade of blue, or you want to increase the font size. The portal uses Twitter Bootstrap for styling; they have great documentation on changing the look and feel of topics using their framework. There’s also a short topic on that in this sample content.

The next common thing is to change strings (from here on out I’ll call them labels) within the portal, things like the portal name or the footer of each page. There’s a topic on how to do that, too. It also tells you how to add labels for other languages.

If you want to modify the portal, or the metadata in the database, you’re going to have to learn the portal code itself. It’s written using Sinatra, a Ruby DSL, with all the content stored in Apache CouchDB. Maybe some day I’ll document some stuff related to that, but until then, read the code, Luke!

If you want to help, send me a push request. Due to the nature of this portal, I may need to ask you to agree to some licensing requirements (yep, I work with lawyers), but don’t let that scare you. I’d appreciate your help.

  • Replace sample content with your own

    Replace the sample content with your own by removing the files in content/en-us/sample, then adding your own content in a language specific subdirectory of content.
  • Change the styling of the portal

    Changing the styling of the portal requires you to add Twitter Bootstrap and Fontawesome as git submodules. Then you update the styles by editing LESS files, which are then compiled to CSS.
  • Change labels in the portal

    All the text in that portal that isn’t in your topics or the table of contents are labels. Updating the text requires updating a label file.
  • Portal Configuration

    There are configuration options that affect the portal.
